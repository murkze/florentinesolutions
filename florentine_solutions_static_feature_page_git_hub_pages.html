How it works

From Upload → Evidence‑Backed Findings

Structured upload

Upload PDFs (drawings, energy calcs, schedules), choose project class (e.g., R1 addition), add site address or parcel ID.

Auto‑enrichment

We fetch zoning, overlays, heritage, floodplain, lot size/frontage, and prior permits (where available) to contextualize checks.

Policy engine

Versioned rulebooks codify municipal code/bylaws. Each rule emits findings with citations, severity, and remediation tips.

Document parser

Plan‑sheet OCR & pattern recognition extracts titles, stamps, schedules, elevations, setbacks, and critical dimensions.

Guardrailed LLM

Models reference authoritative text only with retrieval filters, confidence thresholds, and deterministic fallbacks.

Conflict detection

Cross‑checks site context (lot/overlay) vs proposal (height, coverage, use, parking) to flag inconsistencies early.

PDF pre‑check report

One clean document summarizing passes/fails, missing items, and risks — ready for internal review or to return to the applicant.

Machine‑readable JSON

Every finding encoded with ruleId, evidence, confidence, severity, remediation.

Core Modules

What runs under the hood

Checklist Engine

City‑specific pre‑submission checklists detect missing forms, schedules, stamps, fees, and plan‑set pages.

Versioned per city

Policy Engine

Codified zoning/land‑use/overlay rules with snippet citations and exception handling for variances.

Document Parsing

OCR + layout cues to extract sheet titles, scales, elevations, dimensions, seals, and revision deltas.

GIS Context

Parcel metadata, lot geometry, overlays (flood/heritage/parking), frontage, and adjacent uses.

Quality Gates

Dual‑pass validation (deterministic → model review) with thresholds and human‑in‑the‑loop overrides.

Coverage & Data

Where it works and how it stays current

Municipal scope

Start with target municipalities and expand via a versioned ruleset library that ships with checklists, policy references, document expectations, and exception handling.

Sources & refresh

Bylaws and code references include source URLs, issue dates, and change logs. Scheduled refreshes with reviewed diffs keep the rulebase current.

Data governance

Partner data segregated by tenant. No training on partner‑private uploads unless explicitly allowed. PII redaction for stored artifacts when feasible.

Interoperability

Outputs support CSV, JSON, and PDF. Optional webhooks for partner pipelines. Import adapters for common e‑permit portals to reduce double‑entry.

Accuracy & QA

Explainable by design

Grounded reasoning

Retrieval strictly scoped to verified sources. Each finding cites the exact clause or checklist item used to decide.

Confidence thresholds

Low‑confidence findings are labeled for human review with side‑by‑side evidence. Deterministic rules always take precedence.

Test harness

Golden dataset by city validates the rulebase before releases; regression suite prevents silent drift.

Audit trail

Every run retains version IDs for rulebooks, model prompts, and input artifacts so results are reproducible.

Security

Built for sensitive workloads

Isolation

Tenant‑level data isolation. Artifacts encrypted at rest and in transit. Strict TTLs for transient parsing caches.

Access control

Role‑based access for municipal staff vs. external applicants. Immutable run logs for compliance.
